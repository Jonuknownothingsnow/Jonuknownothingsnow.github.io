零散记录：2020-03-22

Posted on 2020-03-22 Edited on 2020-08-18 In 零散记录

机器学习进阶课程列表 Reddit

reddit的老哥们弄了一个进阶课程的整合贴，相对于大家熟知的CS231n这些课程，列表里的课偏深入以及具体某个细分方面的机器学习课程。

我有一种体验，不只是在算法，整个代码技术领域中，入门的课程，基础的资料，还是挺丰富的，只要真的想学，就一定能找到合适的入门资料，这是挺好的。但是相对进阶一些的问题，其资料的数量就会迅速下降。找起来也要花很多额外的力气。这次reddit老哥整这个列表还是挺好的。

逼真的多角色文本-语音生成器 官网

目前由MIT资助的组织”15”发布了一个文本-语音生成器的demo，在上面可以选择不同的虚拟人物，输入文本来生成语音（目前仅支持英文），当有一些特有名词时，你还可以通过在括号里注明发音方式来保证生成的语音正确。

目前我看到有支持my little pony里的暮光闪闪，传送门里的GLaDos等角色。测试里以下，虽然没法控制语音的情感，生成的语音中也偶尔有些电流的感觉（偶有变形金刚感），但效果已经十分出色。拟真度比谷歌家的还是要差些，但是定制角色这个是在有点实力。

比较遗憾的是目前模型相关内容还没开源，只能先玩玩这个demo app。

Pytorch-Lightning Github

说实话作为一个偏工程的算法老哥，虽然pytorch用起来是很自由很爽，但是有的细节的东西我真的不想每次都手动控制，比如说控制训练的迭代、梯度下降等。有的时候为了实现一些新奇的想法，或许有必要去手动控制，但大部分时间我觉得这些事情框架帮我做了就完事了。

但pytorch的高级封装fast-ai做的好像又太多了，有点把我当傻瓜的样子。

于是我发现了Pytorch-Lightning这个框架，对Pytorch进行了一些包装，继承它的类之后，你只需要定义几个训练相关的方法，比如数据输入、 优化器参数、训练步操作等，就可以直接像keras那样一键训练了。

它同时也支持无缝地切换到GPU和TPU(?)环境。

虽然这个项目还很新，19年三月开始的，但现在的star数已经达到4300+，而且用起来像个插件一样，大部分的操作还是和原生pytorch一致，我觉得还是一个挺有吸引力的框架，后续可以多尝试一下。

识别DeepFake图像准确率达到99% Arxiv

只有魔法才能击败魔法

今日一篇论文提出,可以以高达99%的准确率识别出各种GAN模型生成的伪造图片，群众惊呼只有魔法能够击败魔法。

假的，论坛群众表示只要这玩意的code被放出来，就会有人把它加入到GAN当中，到时候训练出来的图片生成模型只会更强。

太真实了。
