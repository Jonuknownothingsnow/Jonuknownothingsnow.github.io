使用Docker快速搭建环境

假如你在使用GTX1080TI之类的显卡，其实在搭建GPU版的tensorflow还是挺方便的 ，你只需要确认装好了新的显卡驱动，装下docker和nvidia-docker2。然后直接跑官方的docker镜像就好了,注意使用的版本，比如我习惯用python3，我就拉latest-gpu-py3这个标签的镜像：

简单介绍下这句docker的意思

docker run 运行镜像

–runtime=nvidia 使用nvidia-docker2

-u root 使用root用户

-p 8888:8888 将宿主机的8888端口映射给容器的8888端口

-v ~/data:/notebook 将宿主机的~/data目录挂载到容器的/notebook目录上

-it 进入容器

tensorflow/tensorflow:latest-gpu-py3 tensorflow官方镜像的py3 gpu版本。

运行完就可以根据提示进入运行在本地8888端口的jupyter notebook来操作容器了，因为docker其实就像个虚拟机，所以在这里面无论怎么瞎胡搞都无所谓。

Posted on 2018-10-06 Edited on 2020-08-18 In 工具经验

Hello Pandas .2

用了这么久pandas，它真的可以算的上我的吃饭工具了，但是以前一直都很少提及，今天给大家带来一些pandas的略进阶向的小技巧，想要看更基础内容的同学可以看上一篇Hello Pands。

今天给大家带来的是Groupby这个操作，熟悉SQL的同学一定看到这个名字就知道是什么意思了。groupby这个操作，常被称为聚合，这其实来自于是groupby之后的aggregate操作。在平常我们交流的时候，groupby也常常被称为分组，具体叫什么并不重要，重要的是理解groupby这一简单的操作意义就好了。

什么是Groupby

想象一下我们有一群人，他们当中一部分穿着黄色衣服，一部分穿着红色衣服，一部分穿着蓝色衣服，假如我们要计算出穿各种颜色衣服的人各有多少个，那应该怎么做呢？

我们会把人们按他们各自衣服的颜色，分成不同的组，然后数每一个组里的人数。其中我们把人分组的这个动作，就是groupby，其后使用的计算方法，就是aggregate function，在这里是一个count的操作。

BI绝学：指标分析的三套车

冰雪覆盖着伏尔加河，冰河上跑着三套车。

有人在唱着忧郁的歌， 唱歌的是那赶车的人。

最近做BI相关的东西有点多，这两天突然感觉到有点顿悟。来和大家探讨分享一下。我们在平常经遇到的一个问题是，我们有一个很重要的指标，突然有段时间，这个指标跌了很多，或者说出现了异常，我们有没有办法能够快速去定位问题的原因呢，今天我们就来研究这套武功。
